01THE START

为什么尝试 Parent–Child

单层 chunk 始终存在粒度矛盾：短文本更容易精细匹配，长文本更适合作为完整生成上下文。PC0 将现有 chunk 作为 parent，再切成 200 字符、20 overlap 的 child；检索 child，返回 parent。

TOP-3 TOPIC94.3% → 97.1%覆盖提高TOP-1 SOURCE88.6% → 85.7%排序回退
它找到了更多正确主题，却没有更稳定地把正确文档排到前面。
02HYPOTHESIS

child 是否失去了上下文

PC1 只改变 child 的检索文本，在原文前加入确定性结构信息；parent store、child size、FAISS、BM25、RRF 和聚合方式全部保持不变。

Document: {title}
Section: {section_path}
Content: {child content}

判断 / 小评测集中的改善主要由一条 case 驱动。先不宣布成功，反过来检查评测工具本身。

03TURNING POINT

评测集小到无法信任小幅结论

原有 benchmark 只有 35 条有效 case。一条 query 从第 2 名升到第 1 名，就会让 Source MRR@3 变化约 0.0143，足以左右一次小消融。

120 frozen queries34 indexed sources12 indexed topics15 diagnostics excluded
PC1 SOURCE MRR@30.8333 → 0.8389+0.0056TOP-3 SOURCE88.3% → 88.3%持平TOPIC MRR@30.9250 → 0.9167-0.0083

判断 / 结构上下文对部分 query 有帮助，但不是当前排序问题的主要解法。

04DIAGNOSIS

检索、上下文与评测使用了不同粒度

child检索匹配单位→parent上下文载荷单位→source候选名额 · 评测 · provenance

PC0 没有在 source 层去重。同一 Markdown 文档的多个 parent 可以重复占据 Top-3，挤掉其他可能有用的 source。

如果先按 source 分配有限候选位置，再为每个 source 返回其最佳 parent，能否覆盖更多有效来源？
05PC2 · PASSED

Source-Aware Max 两级聚合

NO SOURCE REGRESSION

保留 raw child、FAISS、BM25、child-level RRF 和 candidate_k=80，只改变 RRF 之后的聚合单位。每个 source 只占一个候选位，并返回该 source 中得分最高的 parent。

parent_score = max(child score)
source_score = max(parent score)

06H1 · TRADE-OFF

Max 是否忽略了多次支持

H1 让第二个 child 与第二个 parent 有限贡献；权重在实验前固定，没有根据结果调参。

parent_score = best_child + 0.5 × second_child
source_score = best_parent + 0.3 × second_parent

07REAL BUDGET

检索排序依赖真实上下文预算

context budget+parent length+concatenation order+evidence in prompt

source 命中不等于最佳 parent 属于正确 section，也不等于 evidence span 真正进入 prompt。当前生成按排名拼接文档；预算耗尽后，后排文档可能被截断，因此 Top-1 稳定性仍有实际价值。

08STOP RULE

为什么没有继续搜索权重

停止点来自证据边界，而不是实验未完成。围绕冻结的 120 条 case 继续调整 0.5 / 0.3 容易过拟合；现有标签也无法区分互补证据、overlap 与近重复。

09RESEARCH LENS

PARADE：实验之后获得的解释框架

PARADE 研究 passage-level signal 到 document-level relevance 的聚合。它提供了一个条件性解释：相关性集中在少数 passage 时，max 类方法可能更稳定；独立证据分散时，多 passage 聚合更可能有价值。

USED FOR解释 PC2 / H1 的结果与边界NOT USED FOR决定实验设计或实现学习式 reranker

边界 / 项目没有实现 PARADE，也没有实现学习式 representation aggregation。
